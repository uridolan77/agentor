### SCOPE-001

- **Description:** Policy scope enforcement — Tenant / Workspace / Project rule scoping deferred
- **Evidence / TODO:** `PolicyRuleScope` values (Global, Tenant, Workspace, Project) are modeled on `PolicyRule` and stored correctly. However, `PolicyBundleRulesAdapter.ToProfileRules()` currently treats all rules as globally effective regardless of scope — no identity-scope filtering occurs against run `TenantId` / `WorkspaceId` / `ProjectId`. Scoped policy enforcement requires the adapter to receive run-identity context and filter rules accordingly.
- **Proposed disposition:** **Defer to v1.1** / enterprise policy scoping phase
- **Rationale:** Phase 17 establishes the correct data model. Enforcement is a separate runtime concern that requires run-identity propagation into the evaluation path.
