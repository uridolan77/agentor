# v1.0 release candidate — final verification checklist (PR169)

Run these checks on the **exact commit** (or image built from that commit) you intend to tag as **v1.0.0-rc.1**. Commands assume repository root on a developer workstation with Docker and PowerShell 7+ where noted.

## 1. Build and test

```powershell
dotnet restore Agentor.sln
dotnet build Agentor.sln --no-restore
dotnet test Agentor.sln --no-build
```

Optional Release configuration (matches CI):

```powershell
dotnet build Agentor.sln --no-restore --configuration Release
dotnet test Agentor.sln --no-build --configuration Release
```

## 2. Harness and repository hygiene

```powershell
pwsh ./scripts/verify-harness.ps1 -ExpectedPhase 40 -ExpectedHarnessPass PR170
pwsh ./scripts/verify-repo-clean.ps1
```

## 3. EF Core migrations inventory

```powershell
dotnet ef migrations list --project src/Agentor.Infrastructure/Agentor.Infrastructure.csproj --startup-project src/Agentor.Api/Agentor.Api.csproj --no-build
```

CI runs this with `--configuration Release` after build; align locally if you are validating a Release candidate.

## 4. Docker image

```powershell
docker build -t agentor:rc-verify .
```

## 5. Scripted smoke (local API)

With the API running and configured for your environment:

- **`pwsh ./scripts/smoke.ps1`** — liveness (`GET /health`) as wired in repo.
- **`pwsh ./scripts/release-smoke.ps1`** — broader RC smoke (see `docs/operator/release-smoke.md`).

## 6. Evaluation report slice

```powershell
dotnet test Agentor.sln --no-build --configuration Release --filter "FullyQualifiedName~Agentor.Application.Tests.Evaluation"
```

## 7. Evaluation + performance CI artifacts (local)

```powershell
$env:AGENTOR_EVAL_CI_OUT = "$(Get-Location)/artifacts/evaluation-local"
New-Item -ItemType Directory -Force -Path $env:AGENTOR_EVAL_CI_OUT | Out-Null
pwsh ./scripts/generate-evaluation-ci-artifacts.ps1
```

Review generated files under **`artifacts/evaluation-local`**. Treat CI **`performance-report.*`** rows as **deterministic placeholders** unless you replace them with measured outputs (see `docs/developer/performance-baseline.md`).

## 8. Security checklist

Complete **`docs/security/SECURITY_RELEASE_CHECKLIST.md`** for the target environment (auth mode, OpenAPI gating, secret handling).

## 9. Diagnostics report (authenticated)

With a running host and appropriate credentials, capture **`GET /api/v1/ops/diagnostics-report`** (JSON or `format=markdown`) per `docs/operator/observability.md`. Confirm output contains **no raw secrets** (redaction tests: `ObservabilityRedactionTests`, integration diagnostics assertions).

## 10. Benchmark project compile

```powershell
dotnet build benchmarks/Agentor.Benchmarks/Agentor.Benchmarks.csproj --configuration Release --no-restore
```

## 11. Optional load generation

**`pwsh ./scripts/load-smoke.ps1`** — see `docs/developer/performance-baseline.md` for intent and limitations (local/dev tooling, not an SLO proof).

## Sign-off

Record command output locations and pass/fail in your release ticket. When all mandatory rows pass, the tree is eligible for **`v1.0.0-rc.1`** per `docs/RELEASE/v1.0-RC-TAGGING.md`.
