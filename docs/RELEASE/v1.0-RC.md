# Agentor v1.0 release candidate notes

This document describes **RC readiness** (reviewable, test-gated, documented boundaries). It is **not** a blanket claim of production hardening for every deployment topology.

## Scope

Phase 15 closed platform hardening PR71–PR75 (redaction, benchmarks, CI, migration docs, boundary review). **Phase 36 (PR143–PR148)** adds **release candidate consolidation**: repo truth (`README.md`, `docs/REPO_TRUTH.md`), migration inventory (`docs/developer/MIGRATION_AND_UPGRADE.md`), API contract snapshots (`docs/api/API_CONTRACT_SNAPSHOT.md` + drift test), scripted release smoke (`scripts/release-smoke.ps1`), and a security checklist (`docs/security/SECURITY_RELEASE_CHECKLIST.md`). Harness marker: **phase 36**, **PR148**.

## Quality gates

- `dotnet test Agentor.sln` green (all test projects).
- Docker image builds (`docker build -t agentor:ci .`).
- EF migrations enumerated (`dotnet ef migrations list` in CI).
- Benchmark harness project builds Release (timings are **local**; see `docs/developer/phase15-performance-baselines.md`).
- OpenAPI snapshot drift test (`OpenApiContractSnapshotTests`) green when public routes change intentionally (refresh snapshot per `docs/api/API_CONTRACT_SNAPSHOT.md`).

## Deferred / watchlist

- Items still `passes: false` in `.agentor-harness/feature-list.json` are listed with rationale in `docs/RELEASE/v1.0-RC-DEFERRED-ITEMS.md` and remain **explicitly deferred** until evidenced.

## Next after RC

Post-RC work follows roadmap phases after Phase 36 (only when explicitly scheduled).
